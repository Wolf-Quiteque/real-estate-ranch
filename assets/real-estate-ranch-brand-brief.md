# Real Estate Ranch — Brand Brief for Website Build

> This document contains only the brand material for **Real Estate Ranch** (the community, meetup, education, networking & sponsorship brand). Ranch Capital Lending / Ranch Capital content has been intentionally excluded — this is a standalone community brand website.

---

## 1. Brand Foundation

**Core Brand Idea**
Real Estate Ranch is where real estate investors, entrepreneurs, brokers, sponsors, and business owners come to build relationships, learn the game, and grow together.

It should feel less like a corporate networking group and more like a trusted business circle — a place where people come to meet serious operators and become part of a community that actually helps them move.

**Brand Positioning**
Real Estate Ranch is the investment and business community built for real estate operators who need relationships, education, and deal flow.

**Brand Promise**
We help serious real estate and business operators find the right relationships and the right next move.

**Brand Personality**
- Confident, but not arrogant
- Relationship-driven, but still professional
- Texas/ranch-inspired, but not cheesy cowboy
- Straightforward, but not boring
- Premium, but approachable
- Community-first, but deal-focused

**Brand Archetype**
- Primary: The Connector
- Secondary: The Builder
- Accent: The Trusted Guide

> "We know the game. We know the people. We help you get to the next deal."

## 3. Mission, Vision, Values

To connect real estate investors, business owners, brokers, and partners with the relationships, resources, and education they need to grow with confidence.

5. **Local Trust, Bigger Vision** — Texas-based, relationship-based, but scalable beyond one local meetup.

## 9. Typography

**Direction:** bold, clean sans-serif — Swiss design, not western saloon.

**Font candidates:** Helvetica Neue, Neue Haas Grotesk, Avenir Next, Suisse Int'l, Inter, Manrope, Montserrat.

**Recommended pairing**
- Headlines: Montserrat ExtraBold or Manrope ExtraBold
- Body: Inter Regular
- Accent: Inter Medium / SemiBold

**Type Rules**
- Big headlines, short lines, strong spacing
- ALL CAPS only for short labels
- Avoid script fonts and western novelty fonts
- Avoid clutter

**Scale**
- Hero headline: 64–88px desktop
- Section headline: 36–48px
- Subheading: 22–28px
- Body: 16–18px
- Buttons: 14–16px, uppercase or title case

## 23. Website Build Notes (for Claude Code)

- **Stack:** HTML + Tailwind CSS + vanilla JS (no framework for now)
- **Structure:** Multi-page site (Home, About, Events, Community, Sponsors, Join/Contact) — shared header/footer nav, consistent design system
- **Tone of build:** Modern Ranch — Swiss grid discipline + warm ranch palette + bold editorial typography; avoid literal cowboy clichés (no rope fonts, no saloon signage, no clipart cattle)
- **Must include:** working nav across pages, sponsor logo grid component, event RSVP/contact form (client-side validated, can wire to a backend later), photo/testimonial gallery section, responsive design, subtle scroll/entry animations for a modern feel

### Design Direction: Creative & Engaging (Not Corporate-Boring)

The goal is a site people actually want to scroll through — not a template-looking business page. Build with these techniques:

- **Scroll-triggered animations** — sections fade/slide into view as the user scrolls (Intersection Observer + Tailwind transitions), so the page feels alive rather than static.
- **Bold hero moment** — full-bleed hero image/photo from the meetup, large editorial headline overlay, subtle parallax or slow-zoom effect on the background image.
- **Image-driven storytelling** — use real photos, not stock-photo energy. Every section that can be paired with a photo should be (community, about, events, sponsors). Photos are what make this feel real and inviting, not generic.
- **Micro-interactions** — hover states on cards/buttons (slight lift + shadow + bronze accent), animated underlines on nav links, smooth anchor scrolling.
- **Dynamic photo gallery** — a masonry or auto-scrolling gallery of event photos (community proof) rather than a static grid — this is one of the highest-engagement elements for a community brand.
- **Animated stats/counters** — e.g. "X meetups hosted," "X members," "X sponsors" that count up when scrolled into view — reinforces credibility fast.
- **Sticky/transparent-to-solid nav** — nav bar starts transparent over the hero image, becomes solid black/ivory on scroll.
- **Testimonial carousel** — auto-rotating or swipeable, photo + quote, instead of a static block of text.
- **Generous whitespace + bold type contrast** — per the Swiss-grid direction: let images and headlines breathe rather than cramming content.

### Using the Provided Images

The user has images available in the project to use throughout the site (event/meetup photos, community shots, etc.). Claude Code should:
1. Check the project's image folder (e.g. `/images`, `/assets`, or wherever the files were added) and inventory what's available before building each page.
2. Map images to the sections they best fit: hero background, About page story section, Community/event photo gallery, sponsor section, testimonial photos.
3. Use `object-cover` + fixed aspect-ratio containers so images stay crisp and consistent across screen sizes.
4. Compress/lazy-load images (`loading="lazy"`, appropriately sized) so the added visual richness doesn't hurt performance.
5. If there aren't enough images for every section, prioritize the hero and the Community/Events gallery first — those are the highest-impact spots for this brand.
